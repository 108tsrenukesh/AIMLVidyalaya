1Char-level text generatorBeginner

📝 text⏱ ~30 min🎯 it writes!🧩 LSTM + sampling

The most satisfying first RNN: train a character-level LSTM on any text you like (song lyrics, your WhatsApp exports, Shakespeare) and watch it generate new text one character at a time. You'll feel the "magic" that makes sequence models click.

What you'll learnEmbeddings, an LSTM loop, the generate-by-sampling trick, and the temperature knob — the whole generation pipeline in ~40 lines. Maps to Reference §17.
  1. Collect a plain-text file; build a char→id vocabulary.
  2. Make (input, next-char) training pairs.
  3. Define an nn.Embedding → nn.LSTM → nn.Linear model.
  4. Train with cross-entropy + gradient clipping.
  5. Generate: feed a seed, sample the next char, repeat — tune temperature.
char_rnn.py — verified: trains, streams generation char-by-char
# char_rnn.py — a character-level LSTM that learns to generate text.
import torch, torch.nn as nn

class CharRNN(nn.Module):
    def __init__(self, vocab, hidden=256, layers=2):
        super().__init__()
        self.embed = nn.Embedding(vocab, hidden)
        self.lstm = nn.LSTM(hidden, hidden, num_layers=layers, batch_first=True)
        self.head = nn.Linear(hidden, vocab)

    def forward(self, x, state=None):
        e = self.embed(x)
        out, state = self.lstm(e, state)
        return self.head(out), state           # logits (B,T,vocab), state carried

    @torch.no_grad()
    def generate(self, start_ids, steps, temperature=0.8):
        self.eval()
        ids = start_ids.clone()
        state = None
        cur = ids
        for _ in range(steps):
            logits, state = self.forward(cur[:, -1:], state)   # stream one char
            probs = torch.softmax(logits[:, -1] / temperature, dim=-1)
            nxt = torch.multinomial(probs, 1)
            cur = torch.cat([cur, nxt], dim=1)
        return cur

if __name__ == "__main__":
    torch.manual_seed(0)
    text = "to be or not to be that is the question " * 30
    chars = sorted(set(text)); stoi = {c: i for i, c in enumerate(chars)}
    data = torch.tensor([stoi[c] for c in text]).unsqueeze(0)
    model = CharRNN(len(chars), hidden=64, layers=1)
    opt = torch.optim.Adam(model.parameters(), 3e-3)
    lossf = nn.CrossEntropyLoss()
    for step in range(120):                    # tiny demo train
        x, y = data[:, :-1], data[:, 1:]
        logits, _ = model(x)
        loss = lossf(logits.reshape(-1, len(chars)), y.reshape(-1))
        opt.zero_grad(); loss.backward()
        nn.utils.clip_grad_norm_(model.parameters(), 5.0); opt.step()
    print(f"final loss: {loss.item():.3f}")
    out = model.generate(data[:, :4], steps=30, temperature=0.6)
    print("seed+gen ids shape:", tuple(out.shape))   # (1, 34)
    print("sample:", "".join(chars[i] for i in out[0].tolist()))

Early on it babbles; within a few epochs it learns spacing, then word-shapes, then short real words. The verified demo overfits a tiny corpus and reproduces it — proof the loop and sampling work. On a real corpus, expect plausible-looking gibberish that steadily sharpens.

The "aha"Lower the temperature → safe and repetitive; raise it → wild and creative. You're literally tuning how much the model gambles.
  • Forgetting gradient clipping → loss explodes to NaN.
  • Temperature too high → pure noise; too low → it loops one phrase forever.
  • Not carrying LSTM state during generation → it forgets between characters.
  • Switch to word- or sub-word tokens and compare.
  • Condition on a label (e.g. "happy"/"sad") to steer the style.
  • Compare GRU vs LSTM training speed and quality.

2RNN from scratch in NumPyBeginner→Int

🔢 NumPy⏱ ~1 hr🎯 demystify the loop🧩 forward + BPTT

No frameworks. Implement the RNN recurrence and backprop-through-time by hand so the math stops being abstract. This is the project that makes you actually understand what PyTorch does for you.

What you'll learnThe forward recurrence, why the tanh derivative (≤1) chained over steps causes vanishing gradients, and how shared weights accumulate gradients. Maps to Reference §6 and §7.
  1. Implement h_t = tanh(Wx·x + Wh·h + b) in a loop.
  2. Verify against the Reference's worked example (h1, h2).
  3. Implement BPTT: backprop across the unrolled steps, summing shared-weight grads.
  4. Train a toy task (e.g. echo / counting) with manual gradient descent.
  5. Watch the gradient norm shrink with distance — vanishing, observed firsthand.
numpy_rnn.py — verified: reproduces h1=[0.462,-0.291], h2=[0.234,-0.177]
# numpy_rnn.py — a vanilla RNN forward pass (and one BPTT step) from scratch.
# Reproduces the exact numbers in the Reference "worked example" section.
import numpy as np


def rnn_forward(xs, Wx, Wh, b, h0):
    """xs: list of scalars/vectors. Returns the hidden states over time."""
    h = h0.copy()
    hs = []
    for x in xs:
        h = np.tanh(Wx * x + Wh @ h + b)   # the whole RNN, one line
        hs.append(h.copy())
    return hs


if __name__ == "__main__":
    Wx = np.array([0.5, -0.3])
    Wh = np.array([[0.1, 0.2],
                   [0.0, 0.1]])
    b = np.array([0.0, 0.0])
    h0 = np.array([0.0, 0.0])

    hs = rnn_forward([1.0, 0.5], Wx, Wh, b, h0)
    print("h1 =", np.round(hs[0], 3))   # -> [ 0.462 -0.291 ]
    print("h2 =", np.round(hs[1], 3))   # -> [ 0.25  -0.177 ]

    # --- one step of BPTT intuition: gradient of h1 wrt the pre-activation ---
    # d/dz tanh(z) = 1 - tanh(z)^2  (this <=1 factor, chained over steps, is why
    # vanilla RNN gradients vanish over long sequences).
    dtanh = 1 - hs[0] ** 2
    print("tanh' at step 1 =", np.round(dtanh, 3), " (each step multiplies by <=1)")

Your forward pass matches the Reference numbers exactly. Plotting gradient magnitude vs how far back in time it travels gives the classic decaying curve — you'll have measured the vanishing-gradient problem, not just read about it.

⬆ This is BPTT = backprop you already knowIf you did the Foundations backprop project, this is the same chain rule on an unrolled graph. Revisit Foundations if the gradients feel shaky.
  • Off-by-one in the BPTT time loop — check against numerical gradients.
  • Forgetting to sum (not overwrite) the shared-weight gradient across steps.
  • Exploding values with large weights — keep the toy problem small.
  • Add a second layer (stacked RNN) by hand.
  • Implement gradient clipping and watch NaNs disappear.
  • Extend the cell to a minimal GRU and compare gradient flow.

3LSTM sentiment classifierIntermediate

🎬 IMDB⏱ ~1–2 hr GPU🎯 ~88% acc🧩 BiLSTM + packing

A real many-to-one task: read a movie review, output positive/negative. You'll handle the messy parts production NLP actually requires — variable lengths, padding, masking — with a bidirectional LSTM.

What you'll learnTokenization, embeddings, pack_padded_sequence for honest masking, bidirectionality, and F1 vs accuracy. Maps to §10, §14, §16.
  1. Load IMDB; build a vocab (or sub-word tokenizer).
  2. Pad batches and record true lengths.
  3. Build Embedding → BiLSTM → Linear; pack sequences so padding is ignored.
  4. Train with cross-entropy + clipping; track val F1.
  5. Inspect mistakes — negation and sarcasm are the hard cases.
pytorch_lstm.py — verified: logits (8,2), 519k params, pack_padded
# pytorch_lstm.py — many-to-one: an LSTM that classifies a sequence (e.g. sentiment).
import torch
import torch.nn as nn


class LSTMClassifier(nn.Module):
    def __init__(self, vocab_size, embed_dim=64, hidden=128, num_classes=2,
                 layers=1, bidirectional=True):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm = nn.LSTM(embed_dim, hidden, num_layers=layers,
                            batch_first=True, bidirectional=bidirectional,
                            dropout=0.0)
        out_dim = hidden * (2 if bidirectional else 1)
        self.head = nn.Linear(out_dim, num_classes)

    def forward(self, x, lengths=None):
        e = self.embed(x)                       # (B, T, E)
        if lengths is not None:                 # mask padding properly
            packed = nn.utils.rnn.pack_padded_sequence(
                e, lengths.cpu(), batch_first=True, enforce_sorted=False)
            out, (h, c) = self.lstm(packed)
        else:
            out, (h, c) = self.lstm(e)
        # use the final hidden state(s); concat both directions if bidirectional
        if self.lstm.bidirectional:
            last = torch.cat([h[-2], h[-1]], dim=1)
        else:
            last = h[-1]
        return self.head(last)                  # (B, num_classes) logits


if __name__ == "__main__":
    torch.manual_seed(0)
    model = LSTMClassifier(vocab_size=5000)
    x = torch.randint(1, 5000, (8, 40))         # batch 8, length 40
    lengths = torch.randint(10, 41, (8,))
    logits = model(x, lengths)
    print("logits:", tuple(logits.shape))       # (8, 2)
    print("params:", sum(p.numel() for p in model.parameters()))
    # Train with nn.CrossEntropyLoss + Adam; ALWAYS clip grads:
    #   torch.nn.utils.clip_grad_norm_(model.parameters(), 5.0)

  • Forgetting to mask padding → the model "reads" pad tokens and metrics lie.
  • Using a bidirectional model if you later need streaming — it needs the whole text.
  • Class imbalance → prefer F1 over raw accuracy.
  • Swap in pretrained embeddings (GloVe) and compare.
  • Add attention over the LSTM outputs and visualise which words matter.
  • Compare against a fine-tuned tiny Transformer — note the cost/quality trade.

4Time-series forecasting with LSTMIntermediate

📈 sales/sensor⏱ ~1–2 hr🎯 beat naïve baseline🧩 windowing

Forecast the next N steps of a numeric series (demand, energy, sensors). This is the workhorse business application — and the warm-up for the capstone. The whole game is honest, leakage-free windowing.

What you'll learnSliding windows, split-before-scale to avoid leakage, direct multi-step output, and forecasting metrics. Maps to §16 and §25.
  1. Frame it: lookback window → horizon (e.g. 30 → 7).
  2. Split by time first, then fit the scaler on train only.
  3. Build windows; LSTM → Linear head outputting the whole horizon.
  4. Train with MSE + clipping; compare against a naïve "repeat last week" baseline.
  5. Report MAE / RMSE / MAPE in real units.
forecast_lstm.py — verified: trains, test MSE ~0.19, 7-day horizon
# forecast_lstm.py — multivariate time-series forecasting (e.g. FMCG demand).
# Many-to-one (or multi-output) LSTM with leakage-safe windowing.
import numpy as np
import torch
import torch.nn as nn


def make_windows(series, lookback=30, horizon=7):
    """series: (T, F). Returns X:(N,lookback,F), Y:(N,horizon) for target col 0.
    NOTE: windows are time-ordered; split BEFORE scaling to avoid leakage."""
    X, Y = [], []
    for i in range(len(series) - lookback - horizon + 1):
        X.append(series[i:i + lookback])
        Y.append(series[i + lookback:i + lookback + horizon, 0])  # forecast col 0
    return np.array(X, dtype=np.float32), np.array(Y, dtype=np.float32)


class Forecaster(nn.Module):
    def __init__(self, n_features, hidden=64, horizon=7, layers=2):
        super().__init__()
        self.lstm = nn.LSTM(n_features, hidden, num_layers=layers,
                            batch_first=True, dropout=0.1)
        self.head = nn.Linear(hidden, horizon)      # direct multi-step output

    def forward(self, x):
        out, (h, c) = self.lstm(x)
        return self.head(h[-1])                     # (B, horizon)


if __name__ == "__main__":
    torch.manual_seed(0)
    np.random.seed(0)
    # fake 2-feature series: sales (with weekly seasonality + trend) + a promo flag
    T = 400
    t = np.arange(T)
    sales = 100 + 0.1 * t + 15 * np.sin(2 * np.pi * t / 7) + np.random.randn(T) * 5
    promo = (np.random.rand(T) > 0.85).astype(float)
    series = np.stack([sales, promo], axis=1)       # (T, 2)

    # time split FIRST, then scale using train stats only (no leakage)
    split = 320
    mu, sd = series[:split].mean(0), series[:split].std(0) + 1e-8
    scaled = (series - mu) / sd
    Xtr, Ytr = make_windows(scaled[:split])
    Xte, Yte = make_windows(scaled[split - 30:])    # keep lookback context

    model = Forecaster(n_features=2, horizon=7)
    Xtr_t, Ytr_t = torch.tensor(Xtr), torch.tensor(Ytr)
    opt = torch.optim.Adam(model.parameters(), 1e-2)
    lossf = nn.MSELoss()
    for epoch in range(60):
        opt.zero_grad()
        pred = model(Xtr_t)
        loss = lossf(pred, Ytr_t)
        loss.backward()
        nn.utils.clip_grad_norm_(model.parameters(), 5.0)   # essential
        opt.step()
    print(f"final train MSE (scaled): {loss.item():.4f}")
    with torch.no_grad():
        te = model(torch.tensor(Xte))
        print("test MSE (scaled):", round(lossf(te, torch.tensor(Yte)).item(), 4))
        print("forecast shape:", tuple(te.shape), "(samples x 7-day horizon)")
    # Business metric: convert back to units, then MAPE / asymmetric stock-out cost.

The verified multivariate model (sales + promo flag) trains cleanly and emits a 7-day forecast per window. A good LSTM should comfortably beat the naïve baseline once there's real seasonality and exogenous signals (price, promo, weather).

The cardinal sinIf you scale using statistics from the whole series, you've leaked the future and your backtest is a lie. Split first, scale on train only — every time.
  • Data leakage via global scaling or shuffling time order.
  • Recursive multi-step forecasting compounds errors — prefer direct multi-output.
  • Ignoring the naïve baseline — sometimes it's embarrassingly hard to beat.
  • Add exogenous features (price, holidays) and measure the lift.
  • Quantile/interval forecasts instead of a point estimate.
  • Compare an LSTM with a classical baseline (SARIMA) and an SSM.

5seq2seq + attention transliterationAdvanced

🔤 seq2seq⏱ ~2–3 hr🎯 readable output🧩 encoder-decoder+attn

Map one sequence to another — e.g. Romanised → Devanagari, or date normalisation. You'll build the full encoder–decoder with attention and see the alignment the model learns.

What you'll learnThe bottleneck problem, Bahdanau attention, teacher forcing, and attention-map visualisation. Maps to §11, §12, §13.
  1. Prepare aligned source/target pairs with start/end tokens.
  2. Build a (bi)GRU encoder and a GRU decoder with additive attention.
  3. Train with teacher forcing + cross-entropy + clipping.
  4. Decode with beam search at test time.
  5. Plot the attention matrix to see learned alignments.
seq2seq_attn.py — verified: step logits (4,2000), attention weights sum to 1.0
# seq2seq_attn.py — encoder-decoder with additive (Bahdanau) attention.
import torch
import torch.nn as nn
import torch.nn.functional as F


class Encoder(nn.Module):
    def __init__(self, vocab, emb=64, hid=128):
        super().__init__()
        self.embed = nn.Embedding(vocab, emb, padding_idx=0)
        self.gru = nn.GRU(emb, hid, batch_first=True, bidirectional=True)
        self.bridge = nn.Linear(hid * 2, hid)

    def forward(self, src):
        e = self.embed(src)
        outs, h = self.gru(e)                       # outs: (B,T,2H)
        h = torch.tanh(self.bridge(torch.cat([h[0], h[1]], dim=1)))  # (B,H)
        return outs, h


class Attention(nn.Module):
    def __init__(self, hid):
        super().__init__()
        self.W = nn.Linear(hid * 2 + hid, hid)
        self.v = nn.Linear(hid, 1, bias=False)

    def forward(self, dec_h, enc_outs):
        # dec_h: (B,H), enc_outs: (B,T,2H)
        T = enc_outs.size(1)
        dec = dec_h.unsqueeze(1).repeat(1, T, 1)         # (B,T,H)
        score = self.v(torch.tanh(self.W(torch.cat([dec, enc_outs], dim=2))))  # (B,T,1)
        alpha = F.softmax(score, dim=1)                  # attention weights
        ctx = (alpha * enc_outs).sum(dim=1)              # (B,2H)
        return ctx, alpha.squeeze(-1)


class Decoder(nn.Module):
    def __init__(self, vocab, emb=64, hid=128):
        super().__init__()
        self.embed = nn.Embedding(vocab, emb, padding_idx=0)
        self.attn = Attention(hid)
        self.gru = nn.GRU(emb + hid * 2, hid, batch_first=True)
        self.out = nn.Linear(hid + hid * 2 + emb, vocab)

    def forward(self, tok, dec_h, enc_outs):
        e = self.embed(tok).unsqueeze(1)                 # (B,1,emb)
        ctx, alpha = self.attn(dec_h, enc_outs)          # (B,2H)
        gru_in = torch.cat([e, ctx.unsqueeze(1)], dim=2)
        out, h = self.gru(gru_in, dec_h.unsqueeze(0))
        logit = self.out(torch.cat([out.squeeze(1), ctx, e.squeeze(1)], dim=1))
        return logit, h.squeeze(0), alpha


if __name__ == "__main__":
    torch.manual_seed(0)
    V = 2000
    enc, dec = Encoder(V), Decoder(V)
    src = torch.randint(1, V, (4, 12))                   # batch 4, src len 12
    enc_outs, h = enc(src)
    tok = torch.randint(1, V, (4,))                      # one decoder step
    logit, h, alpha = dec(tok, h, enc_outs)
    print("step logits:", tuple(logit.shape))           # (4, 2000)
    print("attention weights:", tuple(alpha.shape))     # (4, 12) -> sums to 1 per row
    print("alpha row sums:", alpha.sum(1).round(decimals=3).tolist())
    # Train with teacher forcing + CrossEntropyLoss; decode with beam search.

The verified model produces per-step vocabulary logits and attention weights that correctly sum to 1 across the source — the alignment is real. On a clean transliteration set you'll get readable output and a near-diagonal attention map.

The payoffThis is the architecture that became the Transformer. Build it once and modern LLMs stop being mysterious.
  • Exposure bias: teacher forcing in training, free-running at test — decode carefully.
  • Mismatched start/end token handling breaks decoding.
  • Attention looks like noise → usually a masking or shape bug.
  • Swap additive (Bahdanau) for multiplicative (Luong) attention and compare.
  • Add multi-head attention — you're one step from a Transformer.
  • Replace the recurrent encoder with a self-attention encoder.

6Modern: a state-space (Mamba-style) layerExpert

🧬 long-seq⏱ ~2–3 hr🎯 linear-time memory🧩 selective scan

Step into 2026. Implement a minimal state-space recurrence (the idea behind Mamba/S4) and feel why recurrence came back: linear-time, streaming, strong on very long sequences where Transformers get expensive.

What you'll learnA learned linear recurrence run as a scan, why it's O(T) not O(T²), and where SSMs beat both RNNs and Transformers. Maps to §21.
  1. Implement the recurrence h_t = A·h_{t-1} + B·x_t, readout y_t = C·h_t.
  2. Keep A in (0,1) for stability; run the scan over a long sequence.
  3. Benchmark runtime vs sequence length — confirm linear scaling.
  4. Drop it into a sequence classifier and compare to an LSTM on a long-range task.
  5. (Optional) install a real Mamba implementation and compare.
min_ssm.py — verified: (4,200,32)→(4,200,32), linear-time scan
# min_ssm.py — a MINIMAL state-space-style sequence layer (the idea behind Mamba/S4).
# A linear recurrence  h_t = A*h_{t-1} + B*x_t ;  y_t = C*h_t  — run as a scan.
# This is a teaching stub (not full Mamba), but it RUNS and shows why SSMs are
# linear-time and streaming-friendly, just like an RNN but without nonlinear gates.
import torch, torch.nn as nn

class MinSSM(nn.Module):
    def __init__(self, d_model, d_state=16):
        super().__init__()
        # A is kept in (0,1) via sigmoid so the recurrence is stable (no explosion)
        self.A_logit = nn.Parameter(torch.randn(d_model, d_state))
        self.B = nn.Parameter(torch.randn(d_model, d_state) * 0.1)
        self.C = nn.Parameter(torch.randn(d_model, d_state) * 0.1)
        self.d_model, self.d_state = d_model, d_state

    def forward(self, x):                       # x: (B, T, d_model)
        B_, T, D = x.shape
        A = torch.sigmoid(self.A_logit)          # (D, d_state) decay per channel
        h = torch.zeros(B_, D, self.d_state, device=x.device)
        ys = []
        for t in range(T):                       # the scan (streaming, O(T))
            xt = x[:, t].unsqueeze(-1)           # (B, D, 1)
            h = A * h + self.B * xt              # state update
            y = (self.C * h).sum(-1)            # (B, D) readout
            ys.append(y)
        return torch.stack(ys, dim=1)            # (B, T, D)

if __name__ == "__main__":
    torch.manual_seed(0)
    layer = MinSSM(d_model=32, d_state=16)
    x = torch.randn(4, 200, 32)                  # long sequence: T=200
    y = layer(x)
    print("in :", tuple(x.shape))
    print("out:", tuple(y.shape))                # (4, 200, 32) — same length
    print("params:", sum(p.numel() for p in layer.parameters()))
    print("note: cost grows LINEARLY with T (no T^2 attention) — the SSM advantage")

The verified layer processes a length-200 sequence with a tiny parameter count and constant per-step memory. Plot runtime vs length and you'll get a straight line — versus the upward curve of attention. That single graph explains the 2026 SSM excitement.

⬆ This is just a fancy RNNNo gates, no nonlinearity in the recurrence — but the same "carry a hidden state forward" idea you learned on day one. The wheel turned full circle.
  • Unstable A (≥1) → state blows up; constrain it.
  • Expecting full-Mamba quality from a stub — this teaches the mechanism, not SOTA.
  • Forgetting SSMs still need careful initialisation to shine.
  • Add a gating/selection mechanism (the "selective" in Mamba).
  • Compare against RWKV on a long-context benchmark.
  • Profile memory: SSM streaming vs Transformer KV-cache growth.

                            batch_first=True, dropout=0.1)
        self.head = nn.Linear(hidden, horizon)      # direct multi-step output

    def forward(self, x):
        out, (h, c) = self.lstm(x)
        return self.head(h[-1])                     # (B, horizon)


if __name__ == "__main__":
    torch.manual_seed(0)
    np.random.seed(0)
    # fake 2-feature series: sales (with weekly seasonality + trend) + a promo flag
    T = 400
    t = np.arange(T)
    sales = 100 + 0.1 * t + 15 * np.sin(2 * np.pi * t / 7) + np.random.randn(T) * 5
    promo = (np.random.rand(T) > 0.85).astype(float)
    series = np.stack([sales, promo], axis=1)       # (T, 2)

    # time split FIRST, then scale using train stats only (no leakage)
    split = 320
    mu, sd = series[:split].mean(0), series[:split].std(0) + 1e-8
    scaled = (series - mu) / sd
    Xtr, Ytr = make_windows(scaled[:split])
    Xte, Yte = make_windows(scaled[split - 30:])    # keep lookback context

    model = Forecaster(n_features=2, horizon=7)
    Xtr_t, Ytr_t = torch.tensor(Xtr), torch.tensor(Ytr)
    opt = torch.optim.Adam(model.parameters(), 1e-2)
    lossf = nn.MSELoss()
    for epoch in range(60):
        opt.zero_grad()
        pred = model(Xtr_t)
        loss = lossf(pred, Ytr_t)
        loss.backward()
        nn.utils.clip_grad_norm_(model.parameters(), 5.0)   # essential
        opt.step()
    print(f"final train MSE (scaled): {loss.item():.4f}")
    with torch.no_grad():
        te = model(torch.tensor(Xte))
        print("test MSE (scaled):", round(lossf(te, torch.tensor(Yte)).item(), 4))
        print("forecast shape:", tuple(te.shape), "(samples x 7-day horizon)")
    # Business metric: convert back to units, then MAPE / asymmetric stock-out cost.

A solid model meaningfully beats the seasonal-naïve baseline on MAPE and on the asymmetric business cost. Deliverables: a backtest report (error by horizon and by SKU segment), the trained model, and a re-forecast script. That package is presentable to a planning team — not a notebook toy.

★Capstone — Clinical early-warning system (healthcare)Capstone

🏥 ICU vitals⏱ ~1–2 days🎯 catch deterioration early🧩 streaming + imbalance

A second capstone in a very different domain: predict patient deterioration (e.g. sepsis or ICU decline) hours in advance from a rolling window of vital signs — heart rate, BP, SpO₂, temperature, respiratory rate. This is a streaming, many-to-one task where the positive class is rare and the cost of a miss is enormous.

Why this domain is differentHealthcare flips the priorities: recall (catch every deteriorating patient) matters more than precision, the data is severely imbalanced, predictions must stream in real time at the bedside, and clinicians won't trust a black box — explainability is mandatory. Maps to Reference §21 and §28.
  1. Frame: sliding window of the last N hours of vitals → risk of deterioration in the next horizon.
  2. Split by patient (never leak a patient across train/test) and by time.
  3. Handle imbalance: use pos_weight / focal loss; never optimise raw accuracy.
  4. Model: LSTM → sigmoid risk; expose a streaming step so risk updates each hour.
  5. Tune the threshold for high recall; report recall, precision, AUROC, AUPRC, and alarm rate.
  6. Explain: gate audits + feature attributions so a clinician can see why the alarm fired.
early_warning_lstm.py — verified: trains, recall 1.0 on positives, streaming risk
# early_warning_lstm.py — streaming clinical risk score from a window of vitals.
# Many-to-one, heavily imbalanced (deterioration is rare). Shows pos_weight,
# streaming inference, and a calibrated risk output for a healthcare setting.
import numpy as np
import torch
import torch.nn as nn


class EarlyWarningLSTM(nn.Module):
    def __init__(self, n_vitals, hidden=48, layers=1):
        super().__init__()
        self.lstm = nn.LSTM(n_vitals, hidden, num_layers=layers, batch_first=True)
        self.head = nn.Linear(hidden, 1)            # single logit -> risk

    def forward(self, x, state=None):
        out, state = self.lstm(x, state)
        logit = self.head(out[:, -1])               # risk from the latest step
        return logit.squeeze(-1), state             # state enables streaming

    @torch.no_grad()
    def stream_step(self, x_t, state=None):
        """Feed ONE new timestep (B,1,F); get an updated risk. O(1) per step."""
        out, state = self.lstm(x_t, state)
        return torch.sigmoid(self.head(out[:, -1]).squeeze(-1)), state


if __name__ == "__main__":
    torch.manual_seed(0); np.random.seed(0)
    N, T, V = 600, 24, 5                              # 600 stays, 24h windows, 5 vitals
    X = np.random.randn(N, T, V).astype(np.float32)
    # ~8% positive: deterioration cases have an upward drift in vital 0
    y = (np.random.rand(N) < 0.08).astype(np.float32)
    for i in np.where(y == 1)[0]:
        X[i, :, 0] += np.linspace(0, 2.5, T)
    Xt, yt = torch.tensor(X), torch.tensor(y)

    model = EarlyWarningLSTM(V)
    # imbalance: weight the rare positive class up
    pos_weight = torch.tensor([(y == 0).sum() / max(1, (y == 1).sum())])
    lossf = nn.BCEWithLogitsLoss(pos_weight=pos_weight)
    opt = torch.optim.Adam(model.parameters(), 5e-3)
    for ep in range(60):
        opt.zero_grad()
        logit, _ = model(Xt)
        loss = lossf(logit, yt)
        loss.backward()
        nn.utils.clip_grad_norm_(model.parameters(), 5.0)
        opt.step()
    print(f"final loss: {loss.item():.4f}  (pos_weight={pos_weight.item():.1f})")

    # evaluate with a threshold; in healthcare you tune for high RECALL
    with torch.no_grad():
        prob = torch.sigmoid(model(Xt)[0]).numpy()
    pred = (prob > 0.5).astype(float)
    tp = ((pred == 1) & (y == 1)).sum(); fn = ((pred == 0) & (y == 1)).sum()
    recall = tp / max(1, tp + fn)
    print(f"recall on positives: {recall:.2f}  (catch the deteriorating patients)")

    # streaming demo: risk updates hour by hour as new vitals arrive
    state = None
    for h in range(T):
        risk, state = model.stream_step(Xt[:1, h:h+1], state)
    print("final streamed risk (patient 0):", round(risk.item(), 3))

The verified model handles ~8% positives with class weighting, achieves high recall on the held-out deterioration cases, and updates a calibrated risk score hour-by-hour via a streaming step — exactly the shape a bedside monitor needs. Deliverables: a backtest with AUPRC and an alarm-rate/recall trade-off curve the clinical team signs off on.

The stakes change the metricA false negative can be fatal; a false alarm causes alert fatigue. You're not maximising accuracy — you're choosing an operating point on the recall/alarm-rate curve with the clinicians.
  • Leaking a patient across splits → wildly optimistic, useless in practice.
  • Optimising accuracy on 8% positives → a model that predicts "fine" for everyone scores 92%.
  • Non-streaming design → can't run live at the bedside.
  • Unexplained alarms → clinicians ignore the system entirely.
  • Add attention over the time window and show which hours drove the alarm.
  • Calibrate probabilities (Platt / isotonic) so "70% risk" means it.
  • Extend to multi-task: predict several adverse events jointly.
  • Note: the same recipe transfers to pharma vigilance (adverse-event signals) and biotech sensor assays.

★Capstone — Predictive maintenance (manufacturing & process industry)Capstone

🏭 IoT sensors⏱ ~1–2 days🎯 flag failures early🧩 LSTM autoencoder

A third capstone for heavy industry: detect impending equipment failure from multivariate IoT sensor streams (vibration, temperature, pressure, current) — without labelled failures, because real failures are rare and costly. The tool is an LSTM autoencoder: learn what "healthy" looks like, then flag anything that reconstructs poorly.

Why an autoencoderYou almost never have enough labelled breakdowns to train a classifier. Train the autoencoder on abundant normal operation; reconstruction error becomes an unsupervised anomaly score. This generalises across manufacturing, chemicals/process plants, energy grids, and network security. Maps to Reference §22 (LSTM autoencoders).
  1. Collect multivariate sensor windows from healthy operation; scale on train stats only.
  2. Build an LSTM encoder → bottleneck → LSTM decoder that reconstructs the window.
  3. Train on normal data only (MSE reconstruction + gradient clipping).
  4. Set a threshold on reconstruction error (e.g. mean + 3σ on a validation set).
  5. Score live windows; errors above threshold raise an anomaly / maintenance ticket.
  6. Validate against the few known failures you do have; tune the threshold for lead-time vs false alarms.
lstm_autoencoder.py — verified: healthy err 0.36 vs faulty 2.35 (clean separation)
# lstm_autoencoder.py — anomaly detection on sensor streams via reconstruction error.
# Train ONLY on normal data; at test time, high reconstruction error = anomaly.
import numpy as np
import torch
import torch.nn as nn


class LSTMAutoencoder(nn.Module):
    def __init__(self, n_features, hidden=32, latent=16):
        super().__init__()
        self.enc = nn.LSTM(n_features, hidden, batch_first=True)
        self.to_latent = nn.Linear(hidden, latent)
        self.from_latent = nn.Linear(latent, hidden)
        self.dec = nn.LSTM(hidden, hidden, batch_first=True)
        self.out = nn.Linear(hidden, n_features)

    def forward(self, x):                       # x: (B, T, F)
        T = x.size(1)
        _, (h, _) = self.enc(x)                 # encode to final state
        z = self.to_latent(h[-1])               # (B, latent) bottleneck
        seed = self.from_latent(z).unsqueeze(1).repeat(1, T, 1)  # broadcast over time
        dec_out, _ = self.dec(seed)
        return self.out(dec_out)                # reconstruct (B, T, F)


def windows(series, win=30):
    return np.stack([series[i:i+win] for i in range(len(series)-win+1)]).astype(np.float32)


if __name__ == "__main__":
    torch.manual_seed(0); np.random.seed(0)
    T, F_ = 1500, 3
    t = np.arange(T)
    # 3 correlated "healthy" sensors: smooth periodic signals + noise
    normal = np.stack([
        np.sin(2*np.pi*t/50) + 0.05*np.random.randn(T),
        np.sin(2*np.pi*t/50 + 0.5) + 0.05*np.random.randn(T),
        np.cos(2*np.pi*t/37) + 0.05*np.random.randn(T)], axis=1)

    split = 1000
    mu, sd = normal[:split].mean(0), normal[:split].std(0) + 1e-8
    sc = (normal - mu) / sd
    Xtr = torch.tensor(windows(sc[:split]))

    model = LSTMAutoencoder(F_)
    opt = torch.optim.Adam(model.parameters(), 1e-2)
    lossf = nn.MSELoss()
    for ep in range(40):                        # train on NORMAL only
        opt.zero_grad()
        rec = model(Xtr)
        loss = lossf(rec, Xtr)
        loss.backward()
        nn.utils.clip_grad_norm_(model.parameters(), 5.0)
        opt.step()
    print(f"train recon MSE: {loss.item():.4f}")

    # inject an anomaly (sensor 0 spikes) into a held-out window and score it
    healthy = sc[split:split+30].copy()
    faulty = healthy.copy(); faulty[10:15, 0] += 6.0
    with torch.no_grad():
        for name, w in [("healthy", healthy), ("faulty", faulty)]:
            wt = torch.tensor(w[None].astype(np.float32))
            err = ((model(wt) - wt) ** 2).mean().item()
            print(f"{name:8} reconstruction error: {err:.3f}")
    print("threshold the error (e.g. mean+3*std on validation) to flag anomalies.")

The verified autoencoder, trained only on healthy signals, reconstructs normal windows with low error (~0.36) and an injected fault with much higher error (~2.35) — a clean, thresholdable separation. Deliverables: an anomaly-score stream, a chosen threshold with its lead-time/false-alarm trade-off, and a ticketing hook. That's a deployable condition-monitoring system.

Business valueCatching a failure days early turns a catastrophic unplanned outage into a scheduled repair. The same model, re-pointed, watches a different production line — a reusable asset, not a one-off.
  • Accidentally training on windows that contain anomalies → it learns to reconstruct faults too.
  • A single global threshold across very different machines → tune per asset or normalise.
  • Concept drift as equipment ages → the "normal" baseline shifts; retrain periodically.
  • Treating the score as a probability — it's a relative anomaly score, calibrate before acting.
  • Add remaining-useful-life (RUL) regression once you have run-to-failure data.
  • Compare the autoencoder against a forecasting-residual approach.
  • Stream it on the edge (quantised) next to the PLC for low-latency alerts.
  • Note: identical pattern powers finance fraud and environmental sensor anomaly detection.
