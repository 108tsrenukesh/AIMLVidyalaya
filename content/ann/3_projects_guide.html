Setup

Datasets & environment

Minimal setup. Projects 1–2 need only NumPy and a tiny synthetic dataset; Project 3 adds PyTorch and downloads MNIST automatically. A CPU laptop is enough for all three.

One-time environment setup

# create an isolated environment (recommended) python -m venv nn-env source nn-env/bin/activate # Windows: nn-env\Scripts\activate # Projects 1 & 2 need only these pip install numpy matplotlib # Project 3 adds PyTorch (CPU build is fine for MNIST) pip install torch torchvision # PyTorch 2.11+ in 2026

Dataset A — XOR (synthetic)

Four points, the classic non-linearly-separable problem. No download — you generate it in two lines. It is the perfect first target because a single neuron provably cannot solve it, so success proves your hidden layer and backprop actually work.

import numpy as np X = np.array([[0,0],[0,1],[1,0],[1,1]], dtype=float) y = np.array([0, 1, 1, 0]) # XOR targets

Dataset B — MNIST (real)

70,000 handwritten digit images (28×28 grayscale), the "hello world" of deep learning. torchvision downloads and caches it for you. A small MLP reaches ~97–98% test accuracy.

from torchvision import datasets, transforms tf = transforms.Compose([ transforms.ToTensor(), transforms.Normalize((0.1307,), (0.3081,)) # MNIST mean/std ]) train = datasets.MNIST("./data", train=True, download=True, transform=tf) test = datasets.MNIST("./data", train=False, download=True, transform=tf)
Normalize with training statistics only. The MNIST mean (0.1307) and std (0.3081) are computed from the training set and applied to both splits — never compute them over the test set, or you leak. This is the most common subtle setup mistake. (See Pitfalls → data leakage.)

Project 1 · ~1 hour · beginner

Neuron from scratch

Build a single artificial neuron in pure NumPy, implement the four activation functions and their derivatives, and verify those derivatives numerically. This makes the atom of every network concrete.

What you'll build & why

A Neuron class that computes a = σ(wᵀx + b), plus a small library of activations (relu, sigmoid, tanh, gelu) and their derivatives. You will plot each function and gradient-check the derivatives against finite differences.

Why first: the neuron is the unit every architecture is made of. Implementing it — and proving your derivatives are correct — removes all the mystery before you assemble many of them in Project 2. Pairs with Reference → The neuron and Activations.

A Neuron class with a working forward method
Four activations + four derivatives
A gradient-check confirming derivatives to ~1e-6
Plots of each activation and its derivative

Procedure

  1. Implement sigmoid, tanh, relu, and gelu as plain functions of a NumPy array.
  2. Implement each derivative: sigmoid_grad, tanh_grad, relu_grad, gelu_grad.
  3. Write a Neuron class storing w (small random) and b (zero), with a forward(x) returning σ(wᵀx + b).
  4. Gradient-check: for a random z, compare each analytic derivative to (f(z+ε) − f(z−ε)) / 2ε with ε = 1e-5.
  5. Plot each activation and its derivative over [−6, 6] and eyeball the shapes against the reference guide.

Activations + derivatives

import numpy as np def sigmoid(z): return 1/(1+np.exp(-z)) def sigmoid_grad(z): s=sigmoid(z); return s*(1-s) def tanh_grad(z): return 1 - np.tanh(z)**2 def relu(z): return np.maximum(0, z) def relu_grad(z): return (z > 0).astype(float) def gelu(z): return 0.5*z*(1+np.tanh(np.sqrt(2/np.pi)*(z+0.044715*z**3)))

The neuron + a numerical gradient check

class Neuron: def __init__(self, n): self.w = np.random.randn(n)*0.01; self.b = 0.0 def forward(self, x): return sigmoid(np.dot(self.w, x) + self.b) # gradient check: analytic vs finite difference z = np.random.randn(5); eps = 1e-5 num = (sigmoid(z+eps) - sigmoid(z-eps)) / (2*eps) print(np.abs(num - sigmoid_grad(z)).max()) # should be ~1e-10

What success looks like

  • The gradient-check max difference prints around 1e-9 to 1e-10 for every activation. If it is large (>1e-4), your derivative is wrong.
  • Plots match the reference shapes: sigmoid and tanh are S-curves that flatten at the tails; ReLU is a hinge at 0; GELU is a smooth ReLU dipping slightly below zero.
  • Neuron.forward on a 3-vector returns a single number in (0,1).
When the gradient check passes for all four, you have proven your activation calculus is correct — the foundation Project 2's backprop depends on.

Common errors & fixes

Gradient check fails for ReLU at z=0. ReLU is non-differentiable exactly at 0; the finite-difference estimate straddles the kink. Test on values away from 0, and define relu_grad(0)=0 by convention.
Sigmoid overflows for large negative z. np.exp(-z) with very negative z is fine, but very positive z in other formulations overflows. Stick to 1/(1+exp(-z)) and you are safe for the test range.
Shapes mismatch in the dot product. np.dot(w, x) needs w and x the same length. Print w.shape and x.shape if you get a dimension error.

Challenges

  • Add leaky_relu (and its derivative) with a configurable α; show it never fully "dies".
  • Make the neuron classify the AND and OR logic gates by setting weights and bias by hand — then confirm no single neuron can do XOR.
  • Vectorize forward to accept a batch (m × n) and return (m,) in one matrix multiply.

Project 2 · ~2.5 hours · advanced

MLP from scratch on XOR

The centerpiece. Build a 2-layer network with a forward pass, hand-coded backpropagation (BP1–BP4), and a gradient-descent training loop — then watch it learn XOR, which a single neuron cannot. When this works, you understand neural networks.

What you'll build & why

A network: 2 inputs → 4 hidden (ReLU) → 1 output (sigmoid), trained with binary cross-entropy. You implement the forward pass, the full backward pass using the four equations, and the weight-update loop — all in NumPy, no autograd. Then you gradient-check the whole network and train it until it solves XOR.

Why it matters: this is the exact engine inside every deep learning framework, with the cover off. Pairs with Forward propagation, Backpropagation, and Gradient descent.

Forward pass with cached intermediates
Backward pass implementing BP1–BP4 by hand
A whole-network gradient check that passes
XOR solved: all 4 points classified correctly
A plot of the learned non-linear decision boundary

Procedure

  1. Initialize W1 (2×4), b1, W2 (4×1), b2 with He init for the ReLU layer, biases at zero.
  2. Write forward(X) returning the prediction and a cache of z1, a1, z2.
  3. Implement binary cross-entropy loss (with clipping).
  4. Write backward(): start from δ₂ = ŷ − y, apply BP4/BP3 for layer 2, propagate with BP2 (multiply by W2ᵀ and ReLU′), then BP4/BP3 for layer 1.
  5. Gradient-check every weight against finite differences before trusting it.
  6. Loop: forward → loss → backward → update, for a few thousand steps. Print loss every 500 steps.
  7. Plot the decision boundary over the unit square to see the learned non-linear region.

The complete from-scratch MLP

import numpy as np np.random.seed(0) X = np.array([[0,0],[0,1],[1,0],[1,1]], dtype=float) y = np.array([[0],[1],[1],[0]], dtype=float) # He init for the ReLU layer; biases at zero W1 = np.random.randn(2,4)*np.sqrt(2/2); b1 = np.zeros((1,4)) W2 = np.random.randn(4,1)*np.sqrt(2/4); b2 = np.zeros((1,1)) def sigmoid(z): return 1/(1+np.exp(-z)) for step in range(10000): # ---- forward ---- z1 = X@W1 + b1; a1 = np.maximum(0, z1) # ReLU z2 = a1@W2 + b2; y_hat = sigmoid(z2) p = np.clip(y_hat, 1e-9, 1-1e-9) loss = -np.mean(y*np.log(p) + (1-y)*np.log(1-p)) # ---- backward (BP1 -> BP4) ---- dz2 = (y_hat - y) / len(X) # sigmoid + BCE shortcut dW2 = a1.T @ dz2; db2 = dz2.sum(0, keepdims=True) da1 = dz2 @ W2.T dz1 = da1 * (z1 > 0) # BP2: times ReLU' dW1 = X.T @ dz1; db1 = dz1.sum(0, keepdims=True) # ---- update ---- for p_,g_ in [(W1,dW1),(b1,db1),(W2,dW2),(b2,db2)]: p_ -= 1.0 * g_ # lr = 1.0 (fine for this tiny problem) if step % 2000 == 0: print(step, round(loss,4)) print((y_hat > 0.5).astype(int).ravel()) # -> [0 1 1 0]

Gradient check (run before trusting backprop)

# perturb one weight, measure loss change, compare to analytic grad def loss_at(W1,b1,W2,b2): a1=np.maximum(0,X@W1+b1); p=np.clip(sigmoid(a1@W2+b2),1e-9,1-1e-9) return -np.mean(y*np.log(p)+(1-y)*np.log(1-p)) eps=1e-5; i,j=0,0 W1[i,j]+=eps; lp=loss_at(W1,b1,W2,b2); W1[i,j]-=2*eps; lm=loss_at(W1,b1,W2,b2); W1[i,j]+=eps print("numeric grad:", (lp-lm)/(2*eps)) # compare to dW1[i,j]

What success looks like

  • Loss falls from ~0.69 (random) toward ~0.01 over the 10,000 steps.
  • Final predictions thresholded at 0.5 are exactly [0, 1, 1, 0] — XOR solved.
  • The numeric gradient matches your analytic dW1[0,0] to ~5–6 decimal places.
  • The decision-boundary plot shows a curved/region split, not a single straight line — visual proof the hidden layer added non-linearity.
If a single neuron (no hidden layer) is given the same task, it stalls at ~0.69 loss and gets two of four wrong — concrete proof of why depth and non-linearity matter.

Common errors & fixes

Loss stuck at 0.69, never drops. Usually a backprop sign/shape bug or zero gradients. Run the gradient check — it pinpoints the exact weight whose analytic gradient is wrong.
Shape errors in the matrix multiplies. Keep targets as (4,1) not (4,). Print every array's shape through one forward+backward pass; the transpose in a1.T @ dz2 is where most mismatches surface.
All hidden units die (a1 all zeros). Bad init or too-high LR drove every pre-activation negative. Re-seed, confirm He init, or lower the LR. (See Pitfalls → dead ReLUs.)
Loss becomes NaN. You forgot to clip before log, or the LR is far too high. Add the np.clip and reduce the learning rate.

Challenges

  • Generalize to arbitrary layer sizes: store layers in a list and loop the forward/backward instead of hard-coding two layers.
  • Swap sigmoid+BCE for softmax+cross-entropy with 2 output units; confirm you get the same answer and the ŷ − y gradient.
  • Add mini-batching and train on a larger synthetic 2-D dataset (two interleaving moons).
  • Add L2 weight decay to the loss and gradients; observe the effect on the boundary.

Project 3 · ~2.5 hours · intermediate

MLP in PyTorch on MNIST

Now the production way. Rebuild the same kind of network in PyTorch with nn.Module and autograd, train it on real handwritten digits with a proper validation loop, and reach ~97–98% test accuracy. You'll see autograd compute the exact gradients you wrote by hand in Project 2.

What you'll build & why

A 784 → 256 → 128 → 10 MLP that classifies MNIST digits, trained with AdamW, cross-entropy, dropout, and an early-stopping validation loop — the canonical 2026 training setup. The contrast with Project 2 is the lesson: same math, but autograd and nn.Module remove all the manual gradient work, letting you focus on architecture and training.

Pairs with Training in practice and Evaluation.

A clean nn.Module MLP with dropout
Train / validation / test split with DataLoaders
A training loop with the five-step inner cycle
~97–98% test accuracy and a learning-curve plot
Best-checkpoint saving + early stopping

Procedure

  1. Load MNIST via torchvision with the normalize transform; carve a validation split off the training set.
  2. Wrap each split in a DataLoader (shuffle the training loader).
  3. Define the MLP as an nn.Module with Flatten → Linear → ReLU → Dropout → Linear → ReLU → Linear (raw logits out).
  4. Set up nn.CrossEntropyLoss and AdamW(lr=3e-4); move model and data to device.
  5. Train: for each epoch, run the five-step loop over the train loader, then evaluate on validation in eval() + no_grad().
  6. Save the model whenever validation accuracy improves; stop after patience epochs without improvement.
  7. Load the best checkpoint and evaluate once on the test set. Plot train/val loss curves.

Model + training loop

import torch, torch.nn as nn from torch.utils.data import DataLoader, random_split from torchvision import datasets, transforms device = torch.device("cuda" if torch.cuda.is_available() else "cpu") tf = transforms.Compose([transforms.ToTensor(), transforms.Normalize((0.1307,),(0.3081,))]) full = datasets.MNIST("./data", train=True, download=True, transform=tf) train_ds, val_ds = random_split(full, [54000, 6000]) test_ds = datasets.MNIST("./data", train=False, download=True, transform=tf) train_dl = DataLoader(train_ds, batch_size=128, shuffle=True) val_dl = DataLoader(val_ds, batch_size=512) test_dl = DataLoader(test_ds, batch_size=512) class MLP(nn.Module): def __init__(self): super().__init__() self.net = nn.Sequential( nn.Flatten(), nn.Linear(784, 256), nn.ReLU(), nn.Dropout(0.2), nn.Linear(256, 128), nn.ReLU(), nn.Dropout(0.2), nn.Linear(128, 10)) # logits def forward(self, x): return self.net(x) model = MLP().to(device) loss_fn = nn.CrossEntropyLoss() opt = torch.optim.AdamW(model.parameters(), lr=3e-4, weight_decay=1e-2) def accuracy(loader): model.eval(); c=t=0 with torch.no_grad(): for xb,yb in loader: xb,yb=xb.to(device),yb.to(device) c += (model(xb).argmax(1)==yb).sum().item(); t += yb.size(0) return c/t best=0.0; patience=5; wait=0 for epoch in range(30): model.train() for xb,yb in train_dl: xb,yb=xb.to(device),yb.to(device) opt.zero_grad() # 1 loss=loss_fn(model(xb), yb) # 2 + 3 loss.backward() # 4 (autograd = your BP1-BP4) opt.step() # 5 val=accuracy(val_dl) if val>best: best=val; wait=0; torch.save(model.state_dict(), "best.pt") else: wait+=1 print(f"epoch {epoch}: val acc {val:.4f}") if wait>=patience: print("early stop"); break model.load_state_dict(torch.load("best.pt")) print(f"TEST acc {accuracy(test_dl):.4f}") # ~0.97-0.98

What success looks like

~97–98%
MNIST test accuracy (MLP)
<1 min/epoch
on a modern CPU
~10–20
epochs before early stop
  • Validation accuracy climbs quickly past 95% in the first few epochs, then improves slowly.
  • Training and validation loss both fall; if val loss starts rising while train keeps falling, dropout/weight-decay are doing their job or you stop early.
  • The final test accuracy lands around 97–98% — for reference, a CNN would reach ~99%.
You have now built the same network twice — once by hand (Project 2) and once in PyTorch (Project 3) — and gotten consistent, correct learning. That is the whole foundation, proven end to end.

Common errors & fixes

Accuracy stuck near 10% (random). You likely applied softmax before CrossEntropyLoss (it wants logits), or forgot opt.zero_grad(). Remove any softmax in the model; add the zero-grad call.
RuntimeError: expected device cpu/cuda. Model and data are on different devices. Ensure both model.to(device) and xb,yb = xb.to(device), yb.to(device).
Validation accuracy higher than training. Often because dropout is active in training but off in eval — usually fine. If wildly off, check you call model.eval() before validating and model.train() before the next epoch.
Out of memory. Lower the batch size, and make sure validation runs inside torch.no_grad() so the graph isn't stored.

Challenges

  • Add a cosine learning-rate schedule (torch.optim.lr_scheduler.CosineAnnealingLR) and compare convergence.
  • Plot a confusion matrix; find which digit pairs the model confuses most (4/9, 3/5 are common).
  • Replace the MLP with a small CNN and beat 99% — your bridge to the next guide in the curriculum.
  • Add mixed precision with torch.amp and time the speedup on a GPU.

Publish your work

A finished project that no one can see is half a project. Packaging it well is how you turn three exercises into something you can show colleagues, a class, or a hiring manager — and how you make it reproducible for your future self.

1. Put it in a Git repo
One folder per project, plus a top-level README. Initialize with git init, commit early and often. Add a .gitignore for __pycache__/, data/, *.pt, and your virtual environment.
2. Write a README that earns trust
State what each project does, the result it achieves (e.g. "97.8% MNIST test accuracy"), how to run it, and what you learned. A clear README is the single highest-leverage thing you can add. Include one figure — the XOR boundary or the MNIST learning curve.
3. Pin your environment
Freeze dependencies with pip freeze > requirements.txt (or a pyproject.toml). Reproducibility is what separates a script from a project. Set a random seed and note it.
4. Turn the from-scratch work into a notebook
Projects 1–2 shine as a Jupyter/Colab notebook: narrative, math, code, and the gradient-check output inline. Colab also gives free GPUs for Project 3. Notebooks are the natural format for teaching — ideal if you're sharing with a learning group.
5. Write up what you learned
A short post — "I implemented backprop from scratch and here's what finally clicked" — cements your understanding and is genuinely useful to others learning the same thing. Explaining is the strongest test of understanding.

You've built the foundation by hand and in PyTorch. Every architecture from here reuses that exact spine — neuron, activation, loss, forward, backprop, gradient descent — and adds one structural idea. Here's the path, each step its own paired guide in this same design.

✓ Foundations (you are here)
MLP, backprop, training. The universal machinery. You can now read any architecture paper and recognize the common parts.
→ Convolutional Neural Networks (CNNs)
The added idea: convolution + weight sharing for spatial data. Project: beat your MNIST MLP with a CNN (~99%), then classify CIFAR-10. Same training loop, new layer type.
→ Recurrent Networks (RNN / LSTM) — briefly
The added idea: a hidden state carried across time. Learn it mainly to understand sequence modeling and the problem attention was invented to solve. Project: a character-level text generator.
→ Transformers & attention
The added idea: self-attention — every token attends to every other in parallel. The dominant 2026 architecture and the basis of large language models. Project: build a tiny GPT-style model from scratch.
Then: specialize
Fine-tuning pretrained models, deployment (ONNX/TorchServe), and the domain you care about — vision, NLP, or applying nets to tabular/operational problems where you already have expertise.
The one thing to carry forward. Nothing ahead replaces what you just learned — it extends it. When a new architecture looks intimidating, find the neuron, the activation, the loss, and the gradient step inside it. They are always there. See the architecture family map →
