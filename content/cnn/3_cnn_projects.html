1Your first CNNBeginner

📦 Fashion-MNIST⏱ ~10 min on GPU🎯 ~91% accuracy🧩 needs: PyTorch

Train a small CNN to classify 28×28 grayscale clothing images into 10 categories. The goal isn't accuracy — it's seeing the whole pipeline (data → conv layers → pooling → classifier → training loop) work end-to-end in under 40 lines.

What you'll learnHow a Conv2d → ReLU → MaxPool stack feeds a linear classifier, and the canonical PyTorch train/eval loop you'll reuse in every later project.

Maps to Reference §06 (block anatomy) and §15 (training recipe).

  1. Install torch and torchvision; pick CPU or GPU.
  2. Load Fashion-MNIST with transforms.v2; normalise with the dataset mean/std.
  3. Define TinyCNN: two conv blocks, each followed by 2×2 max-pool, then one linear head.
  4. Train 5 epochs with AdamW + CrossEntropyLoss (pass raw logits).
  5. Print test accuracy each epoch and watch it climb past 90%.
Project 1 — Fashion-MNIST (PyTorch 2.11)
# Project 1 — your first CNN on Fashion-MNIST (PyTorch 2.11)
import torch, torch.nn as nn, torch.nn.functional as F
from torch.utils.data import DataLoader
from torchvision import datasets
from torchvision.transforms import v2

device = "cuda" if torch.cuda.is_available() else "cpu"
tf = v2.Compose([v2.ToImage(), v2.ToDtype(torch.float32, scale=True),
                 v2.Normalize([0.2860], [0.3530])])
train = datasets.FashionMNIST("./data", train=True,  download=True, transform=tf)
test  = datasets.FashionMNIST("./data", train=False, download=True, transform=tf)
train_dl = DataLoader(train, 128, shuffle=True)
test_dl  = DataLoader(test, 256)

class TinyCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.c1 = nn.Conv2d(1, 16, 3, padding=1)   # 28x28
        self.c2 = nn.Conv2d(16, 32, 3, padding=1)
        self.fc = nn.Linear(32*7*7, 10)
    def forward(self, x):
        x = F.max_pool2d(F.relu(self.c1(x)), 2)    # -> 14x14
        x = F.max_pool2d(F.relu(self.c2(x)), 2)    # -> 7x7
        return self.fc(x.flatten(1))               # raw logits

model = TinyCNN().to(device)
opt = torch.optim.AdamW(model.parameters(), lr=1e-3)
lossf = nn.CrossEntropyLoss()

for epoch in range(5):
    model.train()
    for x, y in train_dl:
        x, y = x.to(device), y.to(device)
        opt.zero_grad(); lossf(model(x), y).backward(); opt.step()
    model.eval(); correct = total = 0
    with torch.no_grad():
        for x, y in test_dl:
            x, y = x.to(device), y.to(device)
            correct += (model(x).argmax(1) == y).sum().item(); total += y.size(0)
    print(f"epoch {epoch}  test acc {correct/total:.3f}")
# Expect ~0.90-0.92 after 5 epochs.

After 5 epochs you should see test accuracy around 0.90–0.92. The first epoch already clears ~0.86 — CNNs learn fast on easy data.

Success looks likeA steadily rising accuracy with no NaNs. If epoch 0 is already ~0.10 (random) and stuck, you likely applied softmax before the loss or mismatched label dtype.
Watch for
  • Forgetting model.eval() before testing — harmless here (no BN) but a habit to build now.
  • Feeding un-normalised 0–255 pixels — training still works but converges slower.
  • Flattening with the wrong size: after two 2×2 pools, 28→14→7, so the linear layer sees 32×7×7.
  • Add a third conv block and BatchNorm; compare convergence speed.
  • Swap max-pool for a stride-2 conv and see if accuracy holds.
  • Plot the 16 first-layer filters — do any look like edge detectors?

2Convolution from scratchBeginner

🔢 NumPy only⏱ ~20 min🎯 match PyTorch to 1e-12🧩 needs: NumPy, PyTorch

Implement 2-D convolution two ways — the literal nested-loop version and the fast im2col + matrix-multiply version — then prove both agree with PyTorch's F.conv2d. This is the project that makes convolution stop being magic.

What you'll learnThat convolution is just "slide, multiply, sum," and that the fast version is one big matrix multiply (the trick every GPU library uses).

Maps directly to Reference §03, §08 (im2col), and the worked numbers in §18.

  1. Write conv2d_naive with two nested loops over output positions.
  2. Write im2col to unroll every patch into a row, then conv2d_fast as one matmul.
  3. Assert the two agree on random inputs.
  4. Compare to F.conv2d — remember PyTorch also does cross-correlation, so no kernel flip is needed.
  5. Reproduce the §16 example (4×4 input, vertical-edge kernel) and check you get the 2×2 map of −1s.
Project 2 — conv from scratch (verified)
# Project 2 — convolution from scratch, verified against PyTorch
import numpy as np, torch, torch.nn.functional as F

def conv2d_naive(X, K):
    kh, kw = K.shape
    Ho, Wo = X.shape[0]-kh+1, X.shape[1]-kw+1
    out = np.zeros((Ho, Wo))
    for i in range(Ho):
        for j in range(Wo):
            out[i, j] = np.sum(X[i:i+kh, j:j+kw] * K)
    return out

def im2col(X, kh, kw):
    Ho, Wo = X.shape[0]-kh+1, X.shape[1]-kw+1
    cols = np.empty((Ho*Wo, kh*kw)); r = 0
    for i in range(Ho):
        for j in range(Wo):
            cols[r] = X[i:i+kh, j:j+kw].ravel(); r += 1
    return cols, Ho, Wo

def conv2d_fast(X, K):
    cols, Ho, Wo = im2col(X, *K.shape)
    return (cols @ K.ravel()).reshape(Ho, Wo)

rng = np.random.default_rng(0)
X = rng.standard_normal((10, 10)); K = rng.standard_normal((3, 3))
mine = conv2d_fast(X, K)
assert np.allclose(mine, conv2d_naive(X, K))           # two methods agree

# ground truth: PyTorch. Note conv2d does cross-correlation too -> no flip needed.
ref = F.conv2d(torch.tensor(X)[None, None], torch.tensor(K)[None, None])[0, 0].numpy()
print("max abs diff vs PyTorch:", np.abs(mine - ref).max())   # ~1e-12

The script prints a max absolute difference versus PyTorch of about 1e-12 to 1e-16 — i.e. identical up to floating-point noise. If it's larger, you have a real bug.

VerifiedThis exact script runs clean: the from-scratch matmul matches PyTorch to ~8.9e-16.
Watch for
  • Flipping the kernel to match "textbook convolution" — PyTorch does cross-correlation, so flipping makes you disagree.
  • Off-by-one output sizes: valid conv gives H−k+1, not H.
  • Mismatched dtypes (float32 vs float64) inflating the diff — keep both float64 for the comparison.
  • Add stride and padding to both implementations and re-verify.
  • Extend im2col to multi-channel input (C_in stacked into the row).
  • Implement the backward pass (∂L/∂K = X⋆δ) and check it against autograd.

3CIFAR-10, for realIntermediate

🖼 CIFAR-10⏱ ~1–2 hr on GPU🎯 ~93% accuracy🧩 residual net + augmentation

Step up to 32×32 colour images and a real architecture: a mini-ResNet with skip connections, BatchNorm, data augmentation, AdamW, cosine LR, and mixed precision. This is a genuine, competitive training recipe in miniature.

What you'll learnWhy skip connections, augmentation, and a good LR schedule together buy you ~10 accuracy points over the naive net from Project 1.
⬆ Beat the MLP — the point of all thisRun the plain dense network from the Foundations guide on CIFAR-10 first. It stalls around 50–55% — flattening a 32×32×3 image throws away every spatial relationship and explodes the parameter count. This CNN reaches ~93% with fewer parameters. That gap, on the exact same data, is the entire argument for convolution — now you've proven it with your own hands.

Maps to Reference §12 (residuals), §14, and §15.

  1. Set up train/eval transforms — augment train only (random crop + flip).
  2. Build the Res block (two conv-BN, a skip, with a 1×1 projection when shapes change).
  3. Stack stem → 4 residual stages → GAP → linear into ResNetMini.
  4. Train ~50 epochs with AdamW, CosineAnnealingLR, and AMP (see Reference §17 for the loop).
  5. Track train vs test accuracy to watch for overfitting.
Project 3 — mini-ResNet on CIFAR-10
# Project 3 — CIFAR-10 with a residual-style net + augmentation
import torch, torch.nn as nn, torch.nn.functional as F
from torch.utils.data import DataLoader
from torchvision import datasets
from torchvision.transforms import v2

device = "cuda" if torch.cuda.is_available() else "cpu"
MEAN, STD = [0.4914,0.4822,0.4465], [0.2470,0.2435,0.2616]
train_tf = v2.Compose([
    v2.RandomCrop(32, padding=4), v2.RandomHorizontalFlip(),
    v2.ToImage(), v2.ToDtype(torch.float32, scale=True), v2.Normalize(MEAN, STD)])
eval_tf = v2.Compose([v2.ToImage(), v2.ToDtype(torch.float32, scale=True), v2.Normalize(MEAN, STD)])

class Res(nn.Module):
    def __init__(self, cin, cout, stride=1):
        super().__init__()
        self.c1 = nn.Conv2d(cin, cout, 3, stride, 1, bias=False); self.b1 = nn.BatchNorm2d(cout)
        self.c2 = nn.Conv2d(cout, cout, 3, 1, 1, bias=False);      self.b2 = nn.BatchNorm2d(cout)
        self.sc = nn.Sequential()
        if stride != 1 or cin != cout:
            self.sc = nn.Sequential(nn.Conv2d(cin, cout, 1, stride, bias=False), nn.BatchNorm2d(cout))
    def forward(self, x):
        out = F.relu(self.b1(self.c1(x)))
        out = self.b2(self.c2(out))
        return F.relu(out + self.sc(x))            # the skip connection

class ResNetMini(nn.Module):
    def __init__(self, n=10):
        super().__init__()
        self.stem = nn.Sequential(nn.Conv2d(3,64,3,1,1,bias=False), nn.BatchNorm2d(64), nn.ReLU())
        self.layers = nn.Sequential(
            Res(64,64), Res(64,128,2), Res(128,256,2), Res(256,256))
        self.head = nn.Sequential(nn.AdaptiveAvgPool2d(1), nn.Flatten(), nn.Linear(256, n))
    def forward(self, x): return self.head(self.layers(self.stem(x)))

# ... DataLoaders + AdamW + CosineAnnealingLR + AMP loop as in Reference §17.
# Expect ~0.92-0.94 test accuracy after ~50 epochs with this recipe.

Expect roughly 0.92–0.94 test accuracy after ~50 epochs. Without augmentation you'll plateau lower and overfit; without skips, deeper versions get harder to train.

Ablation worth doingTrain once with augmentation off — you'll see train accuracy shoot up while test stalls. That gap is overfitting, made visible.
Watch for
  • Augmenting the test set — metrics become noisy and meaningless.
  • Tiny batch + BatchNorm = unstable stats; keep batch ≥ 64 or switch to GroupNorm.
  • Skipping model.eval() — now it matters: BN will use batch stats and your accuracy will wobble with batch size.
  • LR too high with cosine decay — loss spikes early; do a quick LR range test.
  • Add MixUp/CutMix from transforms.v2 and measure the calibration improvement.
  • Try label smoothing 0.1 and compare confidence histograms.
  • Compare your net's FLOPs/params to torchvision's ResNet-18.

4Transfer learningIntermediate

🗂 your own folders⏱ ~20 min🎯 strong acc, little data🧩 EfficientNet-V2

Take a model pretrained on millions of ImageNet photos and adapt it to your categories from just a few hundred images per class. Freeze the backbone, swap the head, train, then optionally fine-tune the top block at a low LR.

What you'll learnThe single highest-leverage skill in applied vision — and the #1 gotcha (using the backbone's own weights.transforms() preprocessing).

Maps to Reference §13. The primer's "borrow trained eyes" idea, made real.

  1. Arrange data as data/train/<class>/ and data/val/<class>/.
  2. Load EfficientNet_V2_S_Weights.DEFAULT and its bundled preprocess.
  3. Freeze all backbone params; replace the final classifier with one sized to your class count.
  4. Train just the head (fast) until val accuracy plateaus.
  5. Unfreeze the top block, drop LR 10–100×, and fine-tune a few more epochs.
Project 4 — transfer learning (torchvision 0.27)
# Project 4 — transfer learning on your own folders (torchvision 0.27)
import torch, torch.nn as nn
from torchvision import models, datasets
from torchvision.models import EfficientNet_V2_S_Weights
from torch.utils.data import DataLoader

device = "cuda" if torch.cuda.is_available() else "cpu"
weights = EfficientNet_V2_S_Weights.DEFAULT
preprocess = weights.transforms()              # the EXACT preprocessing it expects

# expects data/train/<class>/*.jpg  and data/val/<class>/*.jpg
train_ds = datasets.ImageFolder("data/train", transform=preprocess)
val_ds   = datasets.ImageFolder("data/val",   transform=preprocess)
n_cls = len(train_ds.classes)

model = models.efficientnet_v2_s(weights=weights)
for p in model.parameters():                   # 1) freeze the backbone
    p.requires_grad = False
in_f = model.classifier[1].in_features
model.classifier[1] = nn.Linear(in_f, n_cls)   # 2) new head for YOUR classes
model = model.to(device)

# 3) train the head, then unfreeze top block and fine-tune at low LR
opt = torch.optim.AdamW(model.classifier.parameters(), lr=1e-3, weight_decay=1e-2)
lossf = nn.CrossEntropyLoss(label_smoothing=0.1)
# ... standard train/eval loop. With a few hundred images/class expect strong
# accuracy in a handful of epochs — far better than training from scratch.

With a few hundred images per class you should beat a from-scratch model in a fraction of the epochs — often 90%+ on an easy custom task within minutes. Feature-extraction alone is frequently enough.

Tell-tale of successVal accuracy jumps high within the first epoch or two — the backbone's features are already doing the heavy lifting.
Watch for
  • Using your own normalisation instead of weights.transforms() — silently wrecks transfer.
  • Fine-tuning the whole network at full LR — you'll erase the pretrained knowledge ("catastrophic forgetting").
  • Forgetting the new head is trainable while everything else is frozen — check requires_grad.
  • Too few images and no augmentation — the head overfits instantly.
  • Compare feature-extraction vs full fine-tuning accuracy and training time.
  • Swap in timm and try a ConvNeXt-V2 backbone.
  • Use discriminative learning rates (lower for early layers, higher for the head).

5Grad-CAM: see what it seesAdvanced

🔥 heat-maps⏱ ~30 min🎯 visual debugging🧩 hooks + gradients

Build a Grad-CAM tool that overlays a heat-map showing where a CNN looked when it made a prediction. It's the single best way to catch a model that learned a shortcut (the grass-not-cow problem) before it embarrasses you in production.

What you'll learnHow to grab a layer's activations and gradients with hooks, weight the feature maps by gradient, and ReLU the sum into a "looked here" map.

Maps to Reference §17 (the αkc formula) and the pitfalls in §21.

  1. Load a pretrained ResNet-50 and pick the last conv block as the target layer.
  2. Register a forward hook (to capture activations) and a full backward hook (to capture gradients).
  3. Forward an image, backprop the chosen class score.
  4. Average the gradients per channel → weights; weight the activation maps, sum, ReLU.
  5. Normalise, upsample to image size, and overlay as a heat-map.
Project 5 — Grad-CAM from scratch
# Project 5 — Grad-CAM: a heat-map of where the CNN looked
import torch, torch.nn.functional as F
from torchvision import models
from torchvision.models import ResNet50_Weights

weights = ResNet50_Weights.DEFAULT
model = models.resnet50(weights=weights).eval()
target_layer = model.layer4[-1]                # last conv block

acts, grads = {}, {}
def fwd_hook(m, i, o): acts["v"] = o.detach()
def bwd_hook(m, gi, go): grads["v"] = go[0].detach()
target_layer.register_forward_hook(fwd_hook)
target_layer.register_full_backward_hook(bwd_hook)

def grad_cam(x, class_idx=None):
    logits = model(x)                          # x: (1,3,H,W), preprocessed
    if class_idx is None: class_idx = logits.argmax(1).item()
    model.zero_grad()
    logits[0, class_idx].backward()            # gradient of the chosen class
    A, dA = acts["v"][0], grads["v"][0]        # (C,h,w)
    alpha = dA.mean(dim=(1, 2))                # weight each map by mean gradient
    cam = F.relu((alpha[:, None, None] * A).sum(0))   # weighted sum + ReLU
    cam = cam / (cam.max() + 1e-8)
    return cam.cpu().numpy(), class_idx        # upsample & overlay on the image
# If the heat-map sits on the background, your model learned a shortcut (Reference §19).

On a correctly-learned classifier the hot region sits squarely on the object. The payoff is the failure case: point it at a model that cheated and the heat-map lights up the background instead.

The "aha" momentRun it on your Project 3 or 4 model. If the heat-map ever sits off the object, you've literally seen a spurious correlation.
Watch for
  • Using register_backward_hook (deprecated/buggy) instead of register_full_backward_hook.
  • Forgetting model.zero_grad() — gradients accumulate across calls and your map smears.
  • Targeting too early a layer — low-level layers give blurry, uninformative maps; use the last conv block.
  • Not detaching captured tensors — you'll leak memory across many images.
  • Add Grad-CAM++ for sharper maps on multiple instances.
  • Batch it and build a contact-sheet of heat-maps for your val set.
  • Wire it into Project 6 so every defect prediction ships with an explanation.

6U-Net: segment every pixelAdvanced

🩺 medical imaging⏱ ~60 min🎯 per-pixel masks🧩 encoder–decoder + skips

Step up from "what is in this image?" to "which pixels are the tumour / defect / road?" You'll build a U-Net from scratch — the encoder–decoder-with-skips design from Reference §20 that reuses the transposed convolutions from §10.

What you'll learnWhy segmentation needs a decoder, what skip connections actually preserve, and why pixel-accuracy is a lie for this task (use Dice / IoU instead).

Great public datasets to start: the Oxford-IIIT Pet masks, ISIC skin-lesion, or any Kaggle medical-segmentation set.

  1. Load image+mask pairs; resize to 128×128 or 256×256; normalise images, keep masks as 0/1.
  2. Build the U-Net: 3 down-blocks → bottleneck → 3 up-blocks, concatenating the matching encoder feature map at each up step (the skip).
  3. Train with Dice + BCE loss — Dice handles the heavy background imbalance.
  4. Evaluate with Dice score / IoU, never pixel accuracy.
  5. Overlay predicted masks on the input to eyeball boundary quality.
p7_unet.py — verified: in 128×128 → out 128×128, 1.93M params
# p7_unet.py — U-Net from scratch for binary segmentation (e.g. tumour masks).
# Encoder (down) -> bottleneck -> decoder (up) with SKIP connections.
import torch
import torch.nn as nn
import torch.nn.functional as F


def double_conv(cin, cout):
    return nn.Sequential(
        nn.Conv2d(cin, cout, 3, padding=1), nn.BatchNorm2d(cout), nn.ReLU(inplace=True),
        nn.Conv2d(cout, cout, 3, padding=1), nn.BatchNorm2d(cout), nn.ReLU(inplace=True),
    )


class UNet(nn.Module):
    def __init__(self, in_ch=1, out_ch=1, base=32):
        super().__init__()
        self.d1 = double_conv(in_ch, base)          # encoder
        self.d2 = double_conv(base, base * 2)
        self.d3 = double_conv(base * 2, base * 4)
        self.bott = double_conv(base * 4, base * 8)  # bottleneck
        self.pool = nn.MaxPool2d(2)
        # decoder: transposed conv upsamples, then double_conv on concatenated skip
        self.u3 = nn.ConvTranspose2d(base * 8, base * 4, 2, stride=2)
        self.c3 = double_conv(base * 8, base * 4)
        self.u2 = nn.ConvTranspose2d(base * 4, base * 2, 2, stride=2)
        self.c2 = double_conv(base * 4, base * 2)
        self.u1 = nn.ConvTranspose2d(base * 2, base, 2, stride=2)
        self.c1 = double_conv(base * 2, base)
        self.head = nn.Conv2d(base, out_ch, 1)       # 1x1 -> per-pixel logit

    def forward(self, x):
        s1 = self.d1(x);             x = self.pool(s1)
        s2 = self.d2(x);             x = self.pool(s2)
        s3 = self.d3(x);             x = self.pool(s3)
        x = self.bott(x)
        x = self.c3(torch.cat([self.u3(x), s3], dim=1))   # skip-connection concat
        x = self.c2(torch.cat([self.u2(x), s2], dim=1))
        x = self.c1(torch.cat([self.u1(x), s1], dim=1))
        return self.head(x)          # (N, out_ch, H, W) raw logits


def dice_loss(logits, target, eps=1e-6):
    """Dice handles the class imbalance of segmentation (mask is mostly background)."""
    prob = torch.sigmoid(logits)
    num = 2 * (prob * target).sum(dim=(2, 3)) + eps
    den = prob.sum(dim=(2, 3)) + target.sum(dim=(2, 3)) + eps
    return (1 - num / den).mean()


if __name__ == "__main__":
    net = UNet(in_ch=1, out_ch=1)
    img = torch.randn(2, 1, 128, 128)          # 2 grayscale scans
    mask = (torch.rand(2, 1, 128, 128) > 0.7).float()
    out = net(img)
    print("input :", tuple(img.shape))
    print("output:", tuple(out.shape), "(per-pixel logits, same H/W as input)")
    print("dice loss:", round(dice_loss(out, mask).item(), 4))
    print("params:", sum(p.numel() for p in net.parameters()))
    # Real use: combine Dice + BCE, evaluate with IoU / Dice score, NOT pixel accuracy
    # (background dominates, so accuracy lies — see Reference on evaluation).

A working U-Net on a clean dataset reaches a strong Dice score and produces crisp boundaries thanks to the skip connections. The deliverable is a mask overlay that visibly tracks the object edge.

Definition of doneValidation Dice clears your target and the predicted mask hugs the true boundary — not a blurry blob.
Watch for
  • Reporting pixel accuracy. If 95% of pixels are background, predicting "all background" scores 95% and segments nothing. Track Dice/IoU.
  • Mask/image misalignment after augmentation — apply the same geometric transform to both.
  • Checkerboard artifacts from transposed convs — try resize-then-conv decoders if you see them.
  • Class imbalance — Dice or focal loss beats plain BCE when the target is tiny.
  • Swap the encoder for a pretrained ResNet/EfficientNet backbone (a "U-Net with a backbone").
  • Add deep supervision or attention gates and measure the Dice gain.
  • Try DeepLabV3 (dilated convs) and compare context handling on large objects.

p8_yolo.py — run locally with a webcam
# p8_yolo.py — Real-time webcam object detector with a modern anchor-free YOLO.
# Uses Ultralytics (YOLOv8/v10/v11 share this API in 2026). Run locally with a webcam.
#   pip install ultralytics opencv-python
import cv2
from ultralytics import YOLO


def run_webcam(weights="yolov8n.pt", conf=0.4, cam=0):
    """Anchor-free, one-stage detection -> fast enough for live video.
    yolov8n = 'nano' (fastest, edge-friendly). Swap to yolov8s/m for more accuracy."""
    model = YOLO(weights)              # downloads pretrained COCO weights on first run
    cap = cv2.VideoCapture(cam)
    if not cap.isOpened():
        raise RuntimeError("Could not open webcam")

    while True:
        ok, frame = cap.read()
        if not ok:
            break
        # stream=False returns a Results object per frame; .plot() draws boxes+labels
        results = model.predict(frame, conf=conf, verbose=False)
        annotated = results[0].plot()
        cv2.imshow("YOLO live (press q to quit)", annotated)
        if cv2.waitKey(1) & 0xFF == ord("q"):
            break

    cap.release()
    cv2.destroyAllWindows()


def fine_tune_on_custom_data():
    """Train on YOUR classes with a tiny data.yaml (paths + class names).
    YOLO handles the anchor-free head, augmentation, and the train loop for you."""

8Siamese net: one-shot recognitionExpert

👤 face / signature⏱ ~60 min🎯 verify by distance🧩 metric learning

What if you must recognise someone from a single reference photo, with new identities added all the time? You can't retrain a classifier for every new person. The answer is metric learning: train a Siamese net so that same-identity images land close together in an embedding space and different ones land far apart.

What you'll learnEmbeddings over classification, contrastive/triplet loss, and how "verify by distance + threshold" enables one-shot recognition without retraining.

Try it on Omniglot (the one-shot benchmark), AT&T faces, or signature pairs.

  1. Build a shared embedder CNN that maps an image to an L2-normalised vector.
  2. Form pairs: same-identity (label 1) and different-identity (label 0).
  3. Train with contrastive loss (pull same together, push different apart past a margin).
  4. Enrol one reference embedding per identity.
  5. Verify a new image by its distance to each reference; below threshold ⇒ match.
p9_siamese.py — verified: 128-d embeddings, contrastive loss runs
# p9_siamese.py — Siamese network for one-shot learning / face verification.
# Idea: don't classify into fixed classes. Learn an EMBEDDING where same-identity
# pairs are close and different-identity pairs are far. Then "recognise" by distance.
import torch
import torch.nn as nn
import torch.nn.functional as F


class Embedder(nn.Module):
    """Shared CNN that maps an image -> a small L2-normalised embedding vector."""
    def __init__(self, emb_dim=128):
        super().__init__()
        self.body = nn.Sequential(
            nn.Conv2d(1, 32, 3, padding=1), nn.ReLU(inplace=True), nn.MaxPool2d(2),
            nn.Conv2d(32, 64, 3, padding=1), nn.ReLU(inplace=True), nn.MaxPool2d(2),
            nn.Conv2d(64, 128, 3, padding=1), nn.ReLU(inplace=True), nn.AdaptiveAvgPool2d(1),
        )
        self.fc = nn.Linear(128, emb_dim)

    def forward(self, x):
        x = self.body(x).flatten(1)
        return F.normalize(self.fc(x), p=2, dim=1)   # unit-length embeddings


class Siamese(nn.Module):
    """SAME weights process both images (that's the 'Siamese' twin trick)."""
    def __init__(self, emb_dim=128):
        super().__init__()
        self.embed = Embedder(emb_dim)

    def forward(self, a, b):
        return self.embed(a), self.embed(b)


def contrastive_loss(za, zb, label, margin=1.0):
    """label=1 -> same identity (pull together); label=0 -> different (push apart)."""
    d = F.pairwise_distance(za, zb)
    same = label * d.pow(2)
    diff = (1 - label) * F.relu(margin - d).pow(2)
    return (same + diff).mean()


if __name__ == "__main__":
    net = Siamese(emb_dim=128)
    a = torch.randn(4, 1, 64, 64)
    b = torch.randn(4, 1, 64, 64)
    label = torch.tensor([1., 0., 1., 0.])         # 1=same person, 0=different
    za, zb = net(a, b)
    print("embedding shape:", tuple(za.shape))
    print("pair distances :", [round(x, 3) for x in F.pairwise_distance(za, zb).tolist()])
    print("contrastive loss:", round(contrastive_loss(za, zb, label).item(), 4))
    # Verify (one-shot): enrol ONE photo per person as a reference embedding, then a new
    # photo is "person X" if its distance to X's embedding is below a threshold. No
    # retraining needed to add a new person -> that's the power of metric learning.
    # Modern upgrade: swap contrastive for a TRIPLET loss (anchor/positive/negative).

After training, same-identity pairs show small distances and different pairs large ones. You can add a brand-new identity with a single photo — no retraining — and verify it correctly.

Definition of doneA clear distance gap between same and different pairs, and correct one-shot verification of an identity never seen in training.
Watch for
  • Easy pairs dominate training — use hard-negative mining so the model actually learns.
  • Forgetting to L2-normalise embeddings makes the distance threshold unstable.
  • One fixed threshold rarely fits all conditions — calibrate on a validation set.
  • Data leakage: the same identity must not appear in both train and test splits.
  • Switch contrastive → triplet loss (anchor/positive/negative) and compare.
  • Add an SE attention block (Reference §11) to the embedder and measure the gain.
  • Build a tiny enrol/verify demo app with a webcam capture.

9Capstone: packaging defect detectorCapstone

🏭 real-world⏱ a weekend🎯 deployable model🧩 transfer + Grad-CAM + ONNX

The whole series in one build: a classifier that flags defective product packaging/labels from photos — exactly the kind of supply-chain quality check that runs on a factory line. It combines transfer learning, class-imbalance handling, the right metrics, Grad-CAM for operator trust, and an ONNX export for deployment.

What you'll learnHow the pieces fit into a system: rare-defect imbalance, recall over accuracy, interpretability for trust, and train/serve preprocessing parity.

Pulls together Reference §13, §16, §17, and §23.

  1. Collect/organise images into ok and defect folders (defects will be rare — that's the point).
  2. Transfer-learn an EfficientNet-V2-S head; oversample the defect class with a WeightedRandomSampler.
  3. Evaluate with precision/recall and PR-AUC, not accuracy — and tune the threshold to the cost of a missed defect.
  4. Run Grad-CAM (Project 5) on flagged items so a line operator can see why it flagged.
  5. Export to ONNX and verify the deployed preprocessing matches weights.transforms() exactly.
Project 6 — capstone defect detector
# Project 6 — Capstone: packaging/label defect detector (transfer + Grad-CAM + export)
import torch, torch.nn as nn
from torchvision import models, datasets
from torchvision.models import EfficientNet_V2_S_Weights
from torch.utils.data import DataLoader, WeightedRandomSampler

device = "cuda" if torch.cuda.is_available() else "cpu"
weights = EfficientNet_V2_S_Weights.DEFAULT
preprocess = weights.transforms()

# data/{train,val}/{ok,defect}/*.jpg   (defects are rare -> handle imbalance)
train_ds = datasets.ImageFolder("data/train", transform=preprocess)
counts = torch.bincount(torch.tensor(train_ds.targets))
w = (1.0 / counts.float())[train_ds.targets]            # oversample the rare class
sampler = WeightedRandomSampler(w, len(w), replacement=True)
train_dl = DataLoader(train_ds, batch_size=32, sampler=sampler)

model = models.efficientnet_v2_s(weights=weights)
for p in model.parameters(): p.requires_grad = False
model.classifier[1] = nn.Linear(model.classifier[1].in_features, 2)
model = model.to(device)
opt = torch.optim.AdamW(model.classifier.parameters(), lr=1e-3, weight_decay=1e-2)
lossf = nn.CrossEntropyLoss(label_smoothing=0.05)
# train head -> fine-tune top block -> evaluate with PRECISION/RECALL, not accuracy
# (1% defects => 99% accuracy by predicting "ok" always. Track recall on defects!)

Watch for
  • Reporting accuracy on 1% defects — predicting "ok" always scores 99% and catches zero defects. Track recall.
  • Train/serve skew: a different resize or channel order between training and the line PC silently degrades it (Reference §21).
  • Backgrounds that correlate with defects (one camera station for rejects) — Grad-CAM will expose this; fix the data.
  • No drift monitoring — lighting and packaging change over time; capture hard cases and retrain.
  • Add temperature scaling so the confidence score is trustworthy for thresholding.
  • Quantise to INT8 and measure the speed/accuracy trade for the line PC.
  • Wrap it in a tiny service that returns label + confidence + Grad-CAM image per photo.
