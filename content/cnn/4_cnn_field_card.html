04Init · normalisation · activation pickers

Weight init

  • He / Kaiming for ReLU nets: W~𝒩(0, 2/n_in)
  • Xavier/Glorot only for tanh/sigmoid
  • It's PyTorch's conv default — leave it

Normalisation

  • BatchNorm — batch ≥ 16, plain CNN
  • GroupNorm — tiny batch, detection/seg
  • LayerNorm — ConvNeXt, transformers
  • ⚠ call model.eval() at inference

Activation

  • ReLU — default, fast
  • GELU — modern nets
  • SiLU — EfficientNet
  • LeakyReLU if you see dead units

Why each works: Reference §07.

05Training quick-recipe

Sensible starting point

  • Optimizer: AdamW (or SGD+momentum for big nets)
  • LR: cosine decay + short warmup
  • Mixed precision: autocast + GradScaler
  • Batch: as large as VRAM allows
  • Early-stop on val metric, not loss

Augmentation defaults

  • Flip / crop / colour-jitter (baseline)
  • RandAugment + Mixup/CutMix
  • Albumentations for detection/seg
  • Match geometric aug on image and mask

Transfer-learning decision

  • Small + similar data → freeze, train head
  • More data / different domain → fine-tune all, low LR
  • Always reuse the model's own preprocessing

Full recipe + tuning ladder: Reference §15 · §22 · augmentation §14 · transfer §13.

08Ship-it checklist

Order of operations

  • 1. Correctness + train/serve parity
  • 2. Profile latency / throughput / memory
  • 3. Cheapest win first: FP16 or lighter backbone
  • 4. Quantise / prune only if needed — re-measure

Compression

  • INT8 quant — ~4× smaller, 2–4× faster
  • Pruning — structured = faster anywhere
  • Distillation — best accuracy/FLOP

Export by target

  • NVIDIA → TensorRT
  • Android/edge → TFLite
  • Apple → Core ML · Cross → ONNX

Details: Reference §23.

09Top traps (the ones that actually bite)

  • Forgot model.eval() → BatchNorm uses batch stats at test time → accuracy collapses.
  • Reporting pixel accuracy on segmentation → background dominates → the number lies. Use Dice/IoU.
  • Train/serve skew → different resize, BGR vs RGB, or normalisation between train and deploy.
  • Checkerboard artifacts from naïve transposed convs → prefer resize-then-conv.
  • Data leakage → same identity/patient in train and test → inflated scores.
  • Anchor sizes untuned (anchor-based detectors) → poor recall; or just use anchor-free.
  • LR too high with no warmup on a deep net → diverges in the first epochs.

Full list: Reference §21.
