Which algorithm?

If you…Reach forBecause
want a fast robust baseline, little tuningRandom Forestgood defaults, OOB, never overfits by adding trees
have large data / need speed / heavy tuningLightGBMleaf-wise + GOSS/EFB = fastest at scale
want widest support / conservative / smaller dataXGBoostbattle-tested, level-wise growth resists overfit
have high-card categoricals / small-noisy / want best defaultsCatBoostordered boosting + native categoricals
need the last 1–2% of accuracybake off all three + stackthey're a few lines apart; let CV decide
need to extrapolate / smooth trendnot trees — use linear/GAM/hybridtrees predict averages of seen targets
Decide

Tune in this order

Boosting:

  1. Fix a low learning rate (0.03–0.1); let early stopping choose n_estimators.
  2. Tree complexity — max_depth / num_leaves, min_child_weight / min_child_samples.
  3. Regularisation — reg_lambda (L2), reg_alpha (L1), gamma / l2_leaf_reg.
  4. Stochastic subsampling — subsample (rows), colsample_bytree (features).
  5. Lower LR further + more rounds — final generalisation squeeze.

Random Forest: make n_estimators large → tune max_features (the main lever) → min_samples_leaf / max_depth if noisy. That's it.

Always tune with early stopping inside each trial and search with Optuna (Bayesian) over grid/random.
